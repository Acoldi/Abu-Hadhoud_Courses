- why `delim` is not char??
Yes, that’s correct! When you use the `erase` method to remove a range of elements, it includes the first parameter (the starting iterator) and excludes the second parameter (the ending iterator).

For example, in a `vector`:

```cpp
std::vector<int> myvector = {1, 2, 3, 4, 5, 6, 7, 8};
myvector.erase(myvector.begin() + 3, myvector.begin() + 6); // Removes elements at indices 3, 4, and 5
// myvector now contains: 1, 2, 3, 7, 8
```

And in a `set`:

```cpp
std::set<int> myset = {1, 2, 3, 4, 5, 6, 7, 8};
myset.erase(myset.begin(), myset.find(5)); // Removes elements with values 1, 2, 3, and 4
// myset now contains: 5, 6, 7, 8
```

This behavior is consistent across different containers like `vector`, `set`, and `map`.

string TrimLeft(string text)
{
	int i = 0;
	while (text[i] == ' ')
	{
		i++;
	}
	return text.erase(0, i);
}

string TrimRight(string text)
{
	int i = text.length();
	while (text[i - 1] == ' ')
	{
		i--;
	}
	return text.erase(i); // This function erases from i to the end of the string.
}

string TrimAll(string text)
{
	text = TrimLeft(text);
	text = TrimRight(text);

	return text;
